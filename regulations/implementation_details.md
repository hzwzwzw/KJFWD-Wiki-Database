# 清华大学学生勤工助学大队科技服务分队实施细则

本文档包含[纪律管理办法与津贴发放标准](/regulations/basic_new.md)的详细实施细则。

## 一、替班（T班）相关规则

### 1.1 临时求替（临时qt）的认定

为保证岗位服务质量和值班队员的合理安排，对于临时性质的求替做出如下规定：

#### 1.1.1 基于jcqt时间的认定

**jcqt（今C求替）晚于当日18:30的，属于临时qt，该替班队员工时-0.5h。**

- **说明**：jcqt是指在队员群内发送"今日C楼线下班求别人替班"的请求
- **时间界定**：以发送jcqt消息的时间为准
- **处罚对象**：替班队员（即回复"t"的队员）
- **处罚原因**：临时替班可能影响替班队员的时间安排，给予一定补偿性扣减

#### 1.1.2 基于请假时间的认定

**请假晚于当日18:30的，按以下规则处理：**

1. **18:30 - 19:00 之间请假**：视为临时qt
   - 可以正常请假，但会被记录为临时请假
   - 如找到替班队员，替班队员工时-0.5h

2. **19:00 之后请假**：原则上非紧急理由视为旷岗
   - **紧急理由包括但不限于**：突发疾病、家庭紧急情况、学校突发任务等
   - 非紧急理由（如忘记上班、临时有其他安排等）按旷岗处理
   - 需向分队委员会说明情况

#### 1.1.3 释放时间晚于18:30的情况

**T了班又晚于当日18:30释放的同学，也应视作临时qt。**

- **场景说明**：队员A在群内回复"t"接受替班，但在当日18:30之后才释放该班次（即表示无法替班）
- **处理方式**：如有其他队员接替，该队员按临时qt处理，工时-0.5h
- **群内格式**：应使用【实际临时求替：姓名】的格式说明实际求替情况

**示例**：
```
原始消息：张三发送"jcqt"
李四回复："t"
17:00 李四发现无法替班，在群内说明
18:45 王五回复"t"接替
组长记录：【实际临时求替：王五】，王五工时-0.5h
```

### 1.2 替班格式规范

#### 1.2.1 标准格式

**临时求替情况应使用以下格式在组长通知群中说明：**

```
【实际临时求替：姓名】
```

#### 1.2.2 使用场景

1. 释放时间晚于18:30的情况
2. 多人响应后实际替班人与首次响应人不一致的情况
3. 其他需要特别说明的临时替班情况

#### 1.2.3 记录责任

- **组长职责**：在签到表和组长通知群中如实记录临时替班情况
- **队员职责**：及时在群内说明情况，配合组长记录

### 1.3 晚到/早退的详细规定

#### 1.3.1 基本规则

**晚到/早退 < 15分钟：**
- **不记违纪**，但需在群内说明理由
- 说明格式：简要说明原因即可（如"地铁延误"、"下课晚了"等）
- 组长需在签到表上记录情况

**晚到/早退时间分级：**

| 时长范围 | 考勤记录 | 处罚措施 | 备注 |
|---------|---------|---------|------|
| 0 - 15分钟 | 正常 | 无，但需说明理由 | 不影响工时和积分 |
| [15, 30)分钟 | 迟到/早退 | 扣除1积分/次 | 按实际工作时长计入工时 |
| [30, 90)分钟 | 迟到/早退（半旷） | 扣除当班一半工时+额外0.5工时 | 记为半次旷岗 |
| [90, 180]分钟 | 迟到/早退（旷岗） | 扣除当班所有工时+额外1.0工时 | 记为一次旷岗 |

#### 1.3.2 特殊说明

1. **时间计算**：
   - 晚到：以上班时间（如C楼19:00）为基准
   - 早退：以下班时间（如C楼22:00）为基准

2. **工时计算**：
   - 迟到不可顺延下班时间，需在规定时间下班
   - 按实际在岗时长计入工时（扣除迟到/早退时间后的时长）
   - 额外扣除的工时单独计算

3. **说明义务**：
   - 15分钟以内也需要说明理由，但不影响考勤
   - 超过15分钟的迟到/早退，除了记录考勤外，也应说明原因
   - 无特殊原因的频繁迟到/早退，分队委员会将讨论处理

### 1.4 熊楼（熊知行楼）岗位

**熊知行楼B106岗位（离退休教职工线下服务）同样适用以上时间约束规则。**

- 上班时间：工作日 13:30-16:30
- 临时qt时间界定：晚于当日13:00的jcqt视为临时qt
- 请假时间界定：
  - 13:00 - 13:30 之间请假视为临时qt
  - 13:30 之后请假，原则上非紧急理由视为旷岗

### 1.5 替班响应风险行为

为维护替班机制的公平性，防止使用自动回复脚本等不当行为，详见[《关于替班响应（t班）风险行为的暂行管理规定》](/regulations/substitution_risk.md)。

## 二、事故赔偿流程

### 2.1 流程概述

队员在维修工作中如果出现失误导致来访师生设备损坏或数据丢失，应严格按照以下流程处理：

### 2.2 详细流程

#### 第一步：第一时间报告

**报告对象（同时进行）：**
1. **组长**：当日值班组长
2. **任一队委/负责人**（优先当时正值班的队委/负责人）

**报告内容：**
- 事故发生的时间和地点
- 涉及的设备型号和问题
- 操作过程和出现的问题
- 初步评估的损失情况
- 来访师生的联系方式

**组长职责：**
- 协助队员进行善后工作
- 安抚来访师生情绪
- 协助记录现场情况

**队委/负责人职责：**
- 了解事故具体情况
- 评估问题严重程度
- 初步判定事故责任（队员责任/设备原因/不可抗力等）
- 指导后续处理步骤

#### 第二步：报告财务副

**报告时间**：在第一时间报告后的24小时内

**报告内容：**
- 事故基本情况
- 责任判定结果
- 预估赔偿金额
- 来访师生的诉求

**财务副职责：**
- 启动报销流程
- 协调赔偿事宜
- 指导队员准备相关材料

#### 第三步：撰写事故说明并签字

**事故说明应包含：**
1. **基本信息**
   - 队员姓名、学号、联系方式
   - 事故发生时间、地点
   - 值班组长及其他在场队员

2. **事故经过**（详细描述）
   - 来访师生的设备情况和问题描述
   - 队员的诊断过程
   - 具体操作步骤
   - 问题出现的时刻和现象
   - 事后的应急处理

3. **责任分析**
   - 是否按照操作规范进行
   - 是否提前告知风险并签署协议
   - 问题出现的原因分析
   - 责任归属判定

4. **赔偿方案**
   - 损失评估（设备维修费用/数据恢复费用等）
   - 根据服务协议确定的赔偿比例
   - 具体赔偿金额和方式

5. **后续改进**
   - 个人反思和教训
   - 避免类似问题的措施建议

**签字流程：**
1. 队员本人签字确认
2. 组长签字（确认事故记录准确）
3. **队长签字**（最终审核）
4. 来访师生签字（确认赔偿方案）

### 2.3 赔偿原则

根据《清华大学学生勤工助学大队科技服务分队服务协议》：

1. **队员过失导致的损坏**：分队与来访师生共同承担50%责任
2. **设备原因/不可抗力**：按协议约定处理，分队不承担赔偿责任
3. **未签署服务协议的情况**：按实际情况由分队委员会讨论决定

### 2.4 态度要求

**队员应当：**
- 积极配合调查和处理
- 诚恳向来访师生道歉
- 主动承担应负的责任
- 及时反思和改进

**严禁：**
- 隐瞒事故或推卸责任
- 态度恶劣或不配合处理
- 拒绝承担合理的赔偿责任

**特别提醒**：态度恶劣并因此对分队名誉造成较大影响或产生其他严重后果的，分队委员会同意后，可强制退岗。

### 2.5 事故预防

**队员应当：**
1. 严格遵循[技术与操作规范](/regulations/safeopedia.md)
2. 高风险操作前必须签署服务协议
3. 不确定的操作应及时请教组长或其他有经验的队员
4. 对于超出能力范围的问题，应礼貌拒绝或转介专业维修机构
5. 熟悉[风险机型名单](/regulations/watchout.md)，对高风险机型格外谨慎

## 三、爱心勤工相关规定

### 3.1 背景说明

科技服务分队响应大队号召，参与爱心勤工活动。由于C楼岗位空间限制，需要合理安排参与爱心勤工的队员。

### 3.2 爱心勤工的定义

**爱心勤工**是指队员在正常值班时间内，利用空闲时间进行的勤工助学公益活动，不计入分队工时，但可获得其他形式的认可。

### 3.3 申请和记录流程

#### 3.3.1 申请方式

**线上表格录入**：队员需要参与爱心勤工时，应提前在指定的线上表格中登记。

**登记信息包括：**
- 姓名
- 参与日期和时间
- 爱心勤工内容简述
- 预计占用的位置（如需要）

#### 3.3.2 群内格式

**在队员群或组长通知群中说明时，应使用以下格式：**

```
【爱心勤工：姓名】
```

**示例**：
```
【爱心勤工：张三】今晚会在C楼参与爱心勤工活动
```

### 3.4 位置和人数限制

#### 3.4.1 位置限制原因

- C楼三层南侧吧台空间有限
- 需要保证正常值班队员的工作空间
- 需要保证来访师生的服务体验

#### 3.4.2 限制措施

**具体限制方式由分队委员会根据实际情况确定，可能包括：**

1. **人数限制**：
   - 设定每日爱心勤工人数上限
   - 采取先到先得或预约制度
   - 优先保证正常值班队员的工作需要

2. **位置安排**：
   - 指定爱心勤工的工作区域
   - 不占用正常值班队员的工作位置
   - 如位置不足，需要协调其他场所

3. **时间协调**：
   - 优先安排在来访师生较少的时段
   - 与正常值班工作不冲突

### 3.5 组长职责

**当日值班组长需要：**
1. 核实爱心勤工队员是否已在线上表格登记
2. 合理安排爱心勤工队员的位置
3. 确保不影响正常值班工作
4. 如位置不足，及时与爱心勤工队员沟通调整

### 3.6 与大队的沟通

分队委员会负责与大队沟通协调爱心勤工的相关事宜，包括：
- 位置不足的情况反馈
- 爱心勤工政策的调整建议
- 队员参与情况的统计和汇报

## 四、其他补充说明

### 4.1 文档更新

本实施细则根据分队实际运行情况动态调整，如有更新将及时通知全体队员。

### 4.2 解释权

本实施细则由分队委员会负责解释。

### 4.3 反馈渠道

队员如对实施细则有任何疑问或建议，可以：
1. 向组长或分队委员会反馈
2. 在队员大会上提出讨论
3. 通过队员群提出建议
